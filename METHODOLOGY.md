# 大模型应用开发 · 方法论拆解（无代码版）

> 目标：把"用大模型做出能用的东西"这件事，拆成一套可复用的方法。
> 适用：本机 M4/16G + 免费 Colab/Kaggle。不堆代码，只讲方法、步骤与决策点。
> 原则：先判断要不要动模型权重；能用现成模型套壳解决，绝不训练。

---

## 0. 总方法论：四层能力地图

做 AI 应用不是"选个模型开训"，而是一条从"拿来用"到"动手改"的光谱：

| 层 | 方法核心 | 动权重？ | 你的资源能否练 |
|----|----------|----------|----------------|
| 选方向 | 判断做应用层还是改造层 | — | — |
| ① 应用层 | RAG（检索增强）+ Agent（工具编排） | 否 | 本机 Mac 即可 |
| ② 改造层 | 微调 / LoRA / 蒸馏 / 量化 | 是 | 免费 Colab T4 |
| ③ 作品集 | 把前几项组合成可演示产品 | — | 组合即可 |

**关键判断（避免徒劳）**：
- 能用 prompt + RAG + 工具调用解决的 → 永远先走应用层。
- 只有"现成模型真搞不定"时，才进改造层。
- 绝大多数个人/小团队的价值都出在①，③是把①+②变成求职通行证。

---

## 1. 选方向方法

问自己三个问题，按顺序：
1. **有专属数据吗？** 通用模型不懂你的领域/格式，且 prompt 塞不下 → 倾向改造层。
2. **必须离线/本机/低延迟吗？** 隐私数据不能出本地、或要嵌进 App → 蒸馏个小模型。
3. **成本敏感吗？** 每天海量调用，API 太贵 → 自理小模型。

三条都不沾边 → **先做应用层**。这步零训练、出活最快、最能想清"我到底要模型干嘛"。

---

## 2. 应用层方法一：RAG（检索增强生成）

**一句话方法**：把资料变成"可检索的知识"，回答时先查资料再生成，且限定只基于资料答、带引用。

六步闭环（建好不是终点，要迭代）：

1. **定义知识边界**：先写清这个助手"该懂什么、不该懂什么"。边界越清，答得越准，也越能判断失败。
2. **收集与清洗资料**：统一格式（PDF/网页/笔记转成纯文本）、去噪、去重。垃圾进垃圾出，资料质量决定上限。
3. **切分策略**：把文档切成"块"。块太大→检索不精准；块太小→语义断裂。按语义断句或固定长度，留一点重叠（overlap）防截断。
4. **检索设计**：用户提问时，把问题转成向量去库里比最相似的块；可叠加关键词检索（混合检索）补漏。Top-K 调大覆盖全但易引入噪声，调小更准但可能漏。
5. **生成约束**：把"检索到的块 + 问题"拼成提示，要求模型**只基于资料答、并标注来源**。这一步是 RAG 不胡说的关键。
6. **评估迭代**：用指标量化（忠实度 faithfulness 最重要、答案相关性、上下文精度/召回），看哪步拖后腿，回指第 3、4 步重调。

**练到的市场硬通货**：文档解析、分块策略、Embedding、向量检索、混合检索、评估。

---

## 3. 应用层方法二：Agent（智能体）

**一句话方法**：不让模型一次性答完，而是让它自己规划、调工具、看结果、再决定下一步。

五步：

1. **定义目标与可拆解子任务**：先把"终态"写清（比如"给出含天气和行程时间的方案"），再想它能拆成哪些可独立完成的子任务。
2. **列出可用工具（输入输出契约）**：每个工具只定义清楚"给它什么、它还你什么"（如计算器：输入算式、输出结果；搜索：输入问题、输出摘要）。工具越清晰，模型越会用。
3. **规划机制**：让模型根据当前状态决定"下一步调哪个工具"。可以是简单链式（固定顺序），也可以是模型自决（ReAct 式）。
4. **执行-观察-反思 循环**：模型调工具 → 拿到结果 → 判断够不够 → 不够就再规划再调。这是 Agent 的核心，区别于普通"一次性生成"。
5. **失败兜底与人工接管**：工具报错、模型卡死、超过步数 → 要有降级（返回已知信息 / 求助人工）。没有兜底的 Agent 不能上线。

**练到的市场硬通货**：任务规划、工具编排、多步推理、异常处理——正是市场增速最猛的方向（+418%）。

---

## 4. 改造层方法：微调 / 蒸馏 / 量化

**一句话方法**：当应用层不够时，才动模型权重，把通用模型"适配"到你的场景。

五步决策：

1. **确认必须动权重**：先自问"prompt/RAG 真的不行吗？"。多数时候不行也得先用①验证，避免白训。
2. **构建数据集（质量 > 数量）**：数据的质量决定天花板。可用老师模型生成（数据蒸馏）、或人工标注。覆盖你要的"输入→期望输出"模式，去重去噪。
3. **选方法（决策点）**：
   - **LoRA / QLoRA（推荐）**：只训一小部分参数，省显存省时间，个人玩家首选。
   - **全量微调**：精度高但吃算力，需更强 GPU。
4. **训练-评估-防过拟合**：盯训练曲线，用验证集看是否过拟合（死记训练样本、泛化差）。早停、调学习率、加数据。
5. **量化部署上线**：把模型压到 4-bit / INT8，体积和显存大降，能在本地或低成本服务跑起来、高并发响应。把"模型"变成"能用的服务"。

**练到的市场硬通货**：数据工程、SFT/DPO、训练监控、过拟合处理、推理优化（需求 +92%）、部署工程化。

---

## 5. 作品集方法（决定能不能拿 offer）

- **组合而非堆砌**：把一个真实场景做成产品，同时亮出 RAG + Agent + 微调 + 部署。招聘数据说"2 个互补技能过面试，3 个 + 上线项目基本拿 offer"。
- **可演示 > 完整**：一个能跑、能讲清"我做了什么、为什么"的小产品，胜过一个写了一半的大系统。
- **讲清方法**：面试时讲的不该是"我调了哪个库"，而是"我为什么选 RAG 而不是微调、切分策略怎么定的、评估指标怎么看"。这正是本手册的方法论。

---

## 6. 给你的执行节奏（方法论落地）

1. **先① RAG → Agent**（本机，1~2 月）：最快建立"我会做 AI 应用"的基本盘。
2. **再② 微调 → 量化**（Colab，1~2 月）：补齐高薪的"适配"技能。
3. **最后③ 组合成垂直产品**：这是你的求职通行证。

**不要做的事**：一上来预训练大模型（<20% 的活、需巨头算力）、追大模型参数（先把 0.5B~3B 跑熟）、只看不写（每个方法都要自己跑通一遍、改一遍）。
